# 记忆架构验证问题清单

> 生成时间: 2026-05-03
> 状态: 待修复
> 分类依据: A2 / B / T / C / D / Prompt Layout

---

## A2 层问题

### A2-1: 硬边界约束力不足

**描述:** 明确表达的禁忌/偏好在 Prompt 中注入后，有时仍被历史事件或当前状态的内容冲掉，导致模型违反边界。

**严重程度:** 高

**验证方法:** 在 Redis 中写入 `is_active: true, priority: "critical"` 的边界，然后发送相关话题消息，观察模型是否遵守边界。

**预期行为:** A2 边界必须优先生效，不得被 C 层或 B 层内容覆盖。

---

### A2-2: 硬边界候选提取质量不稳定

**描述:** 总结流程中提取的 `a2_candidates` 可能包含误提取的边界（如一次性抱怨被误判为长期禁忌）。

**严重程度:** 中

**验证方法:** 对话中表达"今天不想聊这个"等一次性拒绝，观察是否被写入 `a2_candidates`。

**预期行为:** 一次性拒绝不进入 A2，或进入后 `is_active: false`。

---

## B 层问题

### B-1: 默认状态值注入过度污染

**描述:** `_parse_current_state()` 在空状态时返回默认值 `平静/中性/闲聊`，与设计目标"空块省略"不一致。

**严重程度:** 中

**验证方法:** 清空 `current_state` Redis key，发送普通闲聊消息，观察 Prompt 中是否注入默认状态。

**预期行为:** 无有效状态时不注入任何 B 层内容，包括默认值。

---

### B-2: current_focus 与 unfinished_items 重复表达

**描述:** 当用户关注事项同时被提取为 `current_focus` 和 `unfinished_items` 时，两者在 Prompt 中可能重复出现。

**严重程度:** 中

**验证方法:** 对话中明确承诺"明天继续聊X"，观察 `current_focus` 和 `unfinished_items` 是否重复注入。

**预期行为:** 跨层去重机制优先保留 `unfinished_items`，避免 `current_focus` 重复表达同一事项。

---

### B-3: 状态抖动

**描述:** 连续围绕同一主题对话时，`emotion` 或 `interaction_mode` 可能每轮变化，而非稳定延续。

**严重程度:** 中

**验证方法:** 连续 5 轮安慰场景，观察状态字段是否保持一致。

**预期行为:** 在无明显变化信号时，状态应稳定延续，不每轮抖动。

---

## T 层问题

### T-1: time node 提前激活导致关注点偏移

**描述:** 提前创建的 time node 在激活前就开始影响模型行为，导致模型过早聚焦于未完成事件。

**严重程度:** 高

**验证方法:** 创建"明天考试"的 time node，在到期前发送普通闲聊消息，观察模型是否主动提起考试。

**预期行为:** 未到期 time node 不影响当前轮回复，只在到期后才激活。

---

### T-2: 双入口问题

**描述:** 总结流程中直接写 `unfinished_items`，而主流程是 `pending → active → route_due_time_nodes`，存在双入口和语义不一致风险。

**严重程度:** 中

**验证方法:** 在总结触发轮观察 `unfinished_items` 写入路径，与 time node到期激活路径是否一致。

**预期行为:** 统一为单一入口或统一契约，避免双入口导致状态不一致。

---

### T-3: 到期激活后背景回忆不足

**描述:** time node 到期后激活到 B 层，但 C 层历史检索未稳定触发，导致跨日跟进缺少背景回忆。

**严重程度:** 高

**验证方法:** 用户提到"明天出成绩"，到期后发送新消息，观察是否触发历史检索。

**预期行为:** 包含"结果/成绩/反馈"等关键词的 time node 到期后，应同时触发 C 层检索。

---

## C 层问题

### C-1: 历史检索触发条件偏粗糙

**描述:** `should_trigger_history_retrieval()` 使用 `split()` 分词对中文效果弱，可能漏触发或误触发。

**严重程度:** 中

**验证方法:** 发送含"上次/之前/后来"等回忆词的消息，观察检索是否稳定触发。

**预期行为:** 显式回忆词应稳定触发检索，中文分词应考虑字符级别或词组匹配。

---

### C-2: 历史检索结果与最近窗口重复

**描述:** `deduplicate_with_recent_window()` 的阈值设置可能导致同一事件在 C 层和 D 层同时出现。

**严重程度:** 中

**验证方法:** 发送与最近 6 条消息内容高度相关的历史话题，观察是否重复注入。

**预期行为:** 高重叠事件应在 C 层抑制，避免与 D 层重复。

---

### C-3: 检索结果质量不稳定

**描述:** 当前检索以 `memory_bank` + Chroma summary fallback 为主，无 rerank，可能导致低相关性结果占据位置。

**严重程度:** 中

**验证方法:** 发送复杂话题消息，观察历史检索结果是否与当前输入高度相关。

**预期行为:** 高相关性、高 importance 的结果优先。

---

**严重程度:** 低

## Prompt Layout 问题

### P-1: 跨层重复注入

**描述:** 同一信息可能在 A2、B、C、D 中同时出现，无统一去重机制。

**严重程度:** 高

**验证方法:** 用户表达禁忌 + 历史事件提及同一话题 + 最近窗口提及同一话题，观察是否三层重复。

**预期行为:** 跨层去重机制优先保留 A2，抑制低层重复。

---

### P-2: 空块处理不一致

**描述:** 不同层的空块处理策略不一致，部分返回默认值，部分省略。

**严重程度:** 中

**验证方法:** 清空各层存储，观察 Prompt 中哪些块被省略、哪些注入默认值。

**预期行为:** 所有层在无有效内容时统一省略，不注入默认占位文本。

---

### P-3: 冲突优先级未强制执行

**描述:** 无显式机制保证 `A2 > B > C > D` 的冲突优先级，依赖模型自行判断。

**严重程度:** 高

**验证方法:** A2 边界与历史事件提供相反建议，观察模型是否优先遵守 A2。

**预期行为:** Payload builder 中强制冲突优先级，A2 内容优先于冲突的低层内容。

---

## Token 问题

**描述:** 验证中观察到 Token 可能随轮次增长超出预期，但未建立稳定监控口径。

**严重程度:** 中

**验证方法:** 运行 `scripts/token_baseline_evaluator.py` 多轮，观察增长趋势。

**预期行为:** 动态注入部分增长可解释，无意外爆发性增长。
